    Landing page

    Technical Description

    The index.html file structures the homepage using semantic HTML elements, ensuring clarity and accessibility. The navigation bar includes both desktop and mobile-friendly elements, with a responsive mobile navigation menu controlled by JavaScript. A carousel (.carousel) dynamically displays climate-related messages, using structured div elements and pagination for user interaction. The hero section employs a full-screen background image with a layered overlay for readability.

    The style.css and index.css files style the page, ensuring a visually appealing and responsive layout. The style.css file sets foundational styles like font selection (font-family: "Plus Jakarta Sans"), resets padding/margins, and applies a sticky navigation bar (nav { position: sticky; top: 0; }). The index.css file refines the homepage, implementing an auto-scrolling carousel, animated navigation bar (nav.scrolled::after), and structured content blocks. Media queries optimize the layout for various screen sizes.

    This page also has a custom build carousel. It has it's own styling, which basically just takes the slider elements and stacked them on top of each other in 3d space using 'z-index'. The carousel functionality is given to it using 3 classes. 'active', 'prev' and 'next'. These classes are applied using the carousel js script.

    Accessibility

    Accessibility is addressed through semantic HTML elements, ensuring proper document structure. Alt attributes are provided for images, making them accessible to screen readers. The navigation includes keyboard-friendly elements (button and a for interactivity), and color contrast is maintained for readability. The mobile menu is ARIA-compliant, ensuring accessibility for visually impaired users.

    User-profile Page

    Technical Description

    The profile section is structured using semantic HTML elements within profile-section, providing a detailed overview of a user's contributions, skills, and volunteer programs. The avatar section uses img inside a div without article to display profile pictures, with a status indicator (::after) indicating online activity. The personal details section is implemented using detail-section, ensuring structured information with labels. The skills section employs a dynamic progress bar with a ::after pseudo-element to visually represent proficiency levels. The badges and volunteer programs sections use flexbox (display: flex; flex-wrap: wrap;) for responsive alignment. Additionally, a contact form is included, utilizing form elements and properly labeled input fields for structured data collection. Styling is managed via CSS media queries to ensure responsiveness. The .profile-section applies paddings and flexbox layouts to create a clean design. The .avatar img is styled with border-radius: 50rem for a circular appearance, and .skill-bar::after dynamically adjusts the width to reflect the userâ€™s skill level. The .availability-grid uses grid-template-columns: repeat(7, 1fr); to create a structured week layout.

    Accessibility

    This profile section enhances accessibility by using semantic HTML elements for proper screen reader navigation. Alternative text (alt="profile-picture") is assigned to images, ensuring users with visual impairments can understand the content. Labels are properly associated with form inputs using label, improving usability for assistive technologies. The high contrast between text and backgrounds ensures readability, and interactive elements like buttons and availability grids use hover and focus states to improve user experience.

    Site-map Page

    Technical Description

    The whole site map is generated from an svg. Svg rectangles and lines were used to generate the map. Almost all of the alignments and scaling was done using svg element attributes. Other than that some css styling was used to add color and hover effects to the links.

    Accessibility

    The links in the map have interactive styling, which makes it more accessible to the user.
